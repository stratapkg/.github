# Contributing to stratapkg

Thanks for your interest. The project is in early development —
contributions of any kind are welcome.

## Ways to contribute

- **New packages** — if you have a use case that fits the stratapkg model
  (a layer on top of Prisma, independent, composable), open an issue to discuss
- **Existing packages** — bug fixes, edge case handling, performance improvements,
  additional backend support (e.g. new cache drivers, new blob providers)
- **Tests** — coverage is sparse right now, any additions help
- **Documentation** — clearer examples, better API docs, real-world usage patterns

## Getting started

1. Open an issue describing what you want to work on
2. Wait for a response before writing large amounts of code —
   the architecture is still evolving
3. Fork, branch, implement, open a pull request

## Code style

- TypeScript: follow the existing conventions, Biome before committing
- Commit messages: short imperative summary, present tense
- Each package is independent — changes to one should not affect others
